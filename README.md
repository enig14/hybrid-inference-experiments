# hybrid-inference-experiments
# 하이브리드 추론 실험 정리 (현재 단계)

> ⚠️ 본 문서는 모바일 환경에서의 하이브리드 추론 실험을 정리한 기록으로,  
> Orin Nano 기반 실험 이전 단계의 관찰과 제약 사항을 정리한 문서입니다.

---

## 1. 프로젝트 배경

최근 모바일 기기에는 CPU, GPU, NPU 등 여러 가속기가 탑재되어 있으며, 이를 활용한 추론 실행이 가능해졌습니다.  
본 프로젝트는 특정 모바일 기기에서 추론 실험을 진행하는 과정에서 시작되었습니다.

초기에는 개별 가속기의 동작 여부와 추론 실행 가능 여부를 확인하는 데 초점을 두었습니다.  
그러나 실험을 진행하면서, 가속기 선택과 실행 방식이 전체 실행 구조에 직접적인 영향을 미친다는 점을 인지하게 되었습니다.

---

## 2. 사전 실험 및 문제 인식

S21+ 및 S25 Ultra 환경에서 추론 실험을 진행한 결과, GPU delegate를 적용한 실행 경로에서는 모델 그래프 전체가 GPU로 위임되어 실행되었으며, 반복 실행 과정에서도 안정적인 추론이 가능했습니다.  
반면, NPU를 포함한 실행 경로는 기기 및 환경에 따라 사용 가능한 범위가 제한적이었습니다.

특히 가속기 할당 방식과 실행 경로를 사용자가 명확히 지정할 수 없어, 가속기 간 분할 처리를 포함한 하이브리드 추론 구조에 대한 실험을 구성하는 것 자체가 사실상 불가능했습니다.  
이로 인해 하이브리드 추론 구조에 대한 실험적 검증 역시 진행할 수 없는 상태였습니다.

---

## 3. 문제 재정의

본 프로젝트의 목적은 NPU와 GPU를 동시에 사용하는 구조를 구현하는 데 그치지 않습니다.  
특정 환경에서 하이브리드 추론을 시도하는 과정에서 발생하는 제약을 정리하고, 이러한 제약이 성능에 어떤 영향을 미치는지를 중심으로 살펴보는 데 초점을 둡니다.

즉, 하이브리드 분할 추론이 이론적으로 가능한 구조인지 여부가 아니라,  
실제 환경에서 실험 자체가 성립 가능한지를 먼저 확인하는 것을 본 단계의 문제로 정의합니다.

---

## 4. 설계 범위 및 접근 방식

본 단계에서는 하이브리드 분할 추론이 유의미한 성능 향상을 제공하는지를 판단하기보다는,  
실험을 구성하고 진행하는 과정에서 드러난 가속기 할당 제약과 실행 오버헤드를 정리하는 데 집중합니다.

성능 향상 여부에 대한 판단은,  
가속기 지정과 실행 경로를 명확히 통제할 수 있는 환경에서의 추가 실험을 통해 다루고자 합니다.

---

## 5. 플랫폼 전환 및 향후 계획

모바일 환경에서는 가속기 지정과 내부 동작을 명확히 확인하는 데 제약이 존재하여,  
하이브리드 분할 추론에 대한 실험을 진행하기 어려웠습니다.

이에 따라 이후 단계에서는 실험 조건을 보다 명확히 설정할 수 있는 환경인  
**Jetson Orin Nano 플랫폼**에서 동일한 주제를 이어서 살펴보고자 합니다.

해당 환경에서는 GPU 실행 경로를 기준으로 추론이 실제 환경에서 어떤 성능을 보이는지를 확인하는 것부터 단계적으로 진행할 예정입니다.
